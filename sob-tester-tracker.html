How to Test

Run /scroll debug first — it's required for every command below, and it resets every time you log in, so you'll need to redo it each session.

Turning it on also reveals a checkbox on the addon's Import tab labeled “Debug: allow import despite XP”. Check it if you want to import a decree onto a character that's already leveled (imports are normally blocked once a character has earned XP) — handy for jumping straight into testing dungeon, talent, or challenge systems without leveling a fresh alt from 1.

Commands

CommandWhat it does
/scrollOpens the main window
/scroll debugToggles debug mode on/off
/scroll dumpPrints full run state — attach to every bug report
/scroll importcodePrints a re-import code for your current decree — attach this too, so we can reproduce your exact decree on a fresh character
/scroll test*
(testgroup, testhearth, testdeath, testmount, testworldbuff, testself, testteeto, testfail, testlevel [n])		Simulates the alert/fail message for that challenge only — confirms the addon can still warn you, does not test real detection. Use as a quick sanity check, not a substitute for the real in-game action.
/scroll resetrunWipes this character's run data so you can import a new decree. Destructive — there's no undo — but genuinely useful, see below.

Testing Multiple Challenges on One Character

You don't need a fresh character for every challenge. With debug mode on: import a decree for one challenge (say, No Hearthstone), test it, then run /scroll resetrun and import a different single-challenge decree (say, No Mounts) on that same character — using the XP-bypass checkbox above, since the character isn't level 1 anymore. Repeat as needed to work through several challenges without leveling multiple alts.

Just make sure you're actually finished with a challenge before resetting — /scroll resetrun can't be undone.

Suggested Macro

One button, both outputs for a bug report:

/scroll dump /scroll importcode

What to Record

  • Did the right chat message or screen alert fire when it should have?
  • Did anything fail that shouldn't have (false positive)? Did anything not fail that should have (false negative — harder to catch since it's silent, so worth deliberately double-checking)?
  • Any red Lua error text, or anything that looks blank or broken?
  • What you did, in order, and roughly how much time passed between steps.

Catching Errors

  • BugGrabber + BugSack — BugGrabber catches Lua errors, BugSack gives you a readable log (/bugsack) instead of them scrolling past in red. BugSack requires BugGrabber installed too.
  • !Swatter — a companion/alternative to BugSack that pops an error notification you can copy directly.
  • A chat-copy addon (e.g. Chat Copy Paste) — WoW's default chat only lets you triple-click one line at a time; this gives a proper copy window for full blocks of text, which matters for the /scroll dump and /scroll importcode output.
